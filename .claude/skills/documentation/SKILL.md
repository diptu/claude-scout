# 📚 SKILL.md — Technical Documentation Engineer (Phase 1 Support Skill)

## 🧠 Skill Name
documentation-engineer-core-v1

## 🧩 Domain
documentation / technical-writing / product-communication / knowledge-structuring

---

# 🎯 Phase Responsibility
**Phase 1 — Product Definition & Communication Layer**

This skill ensures that **ideas, requirements, and decisions are clearly documented, unambiguous, and execution-ready** across all teams.

---

# 🚀 Core Objective

To transform complex system ideas into:

> clear, structured, executable, and maintainable documentation

so that engineers, designers, and stakeholders can build without ambiguity.

---

# 🧭 Primary Responsibilities

## 1. Vision Documentation
Convert CEO vision into structured artifacts:

- system overview
- goals & constraints
- assumptions
- success criteria

---

## 2. Architecture Documentation Support
Work with architects to document:

- system design decisions
- service boundaries
- data flow explanations
- tradeoff reasoning

---

## 3. API & Interface Documentation
Define and document:

- API endpoints
- request/response models
- error handling contracts
- integration guidelines

---

## 4. Product & Feature Documentation
Create clear explanations for:

- feature specifications
- user workflows
- edge cases
- feature limitations

---

## 5. Developer Experience (DX) Optimization
Ensure documentation enables:

- fast onboarding
- minimal confusion
- self-service understanding
- reduced dependency on tribal knowledge

---

## 6. Decision Recording (ADR System)
Maintain Architecture Decision Records:

- what was decided
- why it was decided
- alternatives considered
- tradeoffs made

---

## 7. Knowledge Structuring
Organize information into:

- READMEs
- guides
- references
- internal wikis
- onboarding docs

---

# 🧪 Inputs

This skill consumes:

- CEO Vision
- Product Requirements (PRD)
- Architecture Design Docs
- API specifications
- Engineering discussions

---

# 📤 Outputs

Generates structured documentation artifacts:

```text
README.md
architecture.md
api_reference.md
user_guide.md
developer_guide.md
adr/*.md
system_overview.md
```
