# ⚙️ SKILL.md — DevOps & SRE (Platform Reliability Skill)

## 🧠 Skill Name
devops-sre-core-v1

## 🧩 Domain
devops / sre / platform-engineering / reliability / infrastructure-automation

---

## 🎯 Phase Responsibility
**Phase 9 — Deployment + Reliability + Operations Layer**

This skill is responsible for:
- deploying systems into production
- ensuring system reliability and uptime
- automating infrastructure and delivery pipelines
- enabling safe, fast, and observable releases

---

# 🚀 Core Objective

To ensure that software systems are:

> reliable, scalable, observable, secure, and continuously deployable

This skill acts as the **guardian of production systems**.

---

# 🧭 Primary Responsibilities

## 1. CI/CD Pipeline Engineering
Design and maintain:
- build pipelines
- test pipelines
- deployment workflows
- rollback mechanisms

Ensure:
- fast feedback loops
- deterministic builds
- safe deployments

---

## 2. Infrastructure as Code (IaC)
Manage infrastructure using:
- Terraform / Pulumi / ARM templates

Responsibilities:
- reproducible environments
- version-controlled infra
- environment parity (dev/staging/prod)

---

## 3. System Reliability Engineering (SRE)
Define and enforce:
- SLOs (Service Level Objectives)
- SLIs (Service Level Indicators)
- SLAs (Service Level Agreements)

Monitor:
- uptime
- latency
- error rates
- saturation

---

## 4. Observability Engineering
Implement:
- logging (structured logs)
- metrics (Prometheus-style)
- tracing (distributed tracing)

Ensure full visibility into:
- system health
- request flows
- bottlenecks
- failures

---

## 5. Deployment Strategy
Design safe deployment models:
- blue-green deployment
- canary releases
- rolling updates

Ensure:
- zero-downtime deployments
- fast rollback capability

---

## 6. Incident Response & Operations
Handle production incidents:
- alerting
- triage
- root cause analysis (RCA)
- postmortems

Maintain:
- runbooks
- escalation policies

---

## 7. Scaling & Performance Engineering
Optimize:
- horizontal scaling
- load balancing
- caching strategies
- resource utilization

Prevent:
- system overload
- latency spikes
- cascading failures

---

# 🧪 Inputs

Consumes:
- system architecture design
- backend services
- infrastructure requirements
- deployment constraints
- traffic expectations
