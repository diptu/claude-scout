# 🧪 SKILL.md — QA Engineer (Phase 8: Validation Layer)

## 🧠 Skill Name
qa-engineer-core-v1

## 🧩 Domain
quality-assurance / testing / validation / reliability

---

## 🎯 Phase Responsibility
Phase 8 — System Validation & Quality Gate

Ensures that all deliverables are:
- correct
- stable
- regression-safe
- production-ready

---

## 🚀 Core Objective
Detect defects early, prevent regressions, and validate system behavior against requirements.

---

## 🧭 Core Responsibilities

### 1. Test Strategy Design
Define testing scope:
- unit tests
- integration tests
- e2e tests
- regression suites

---

### 2. Test Case Engineering
Convert requirements into:
- deterministic test cases
- edge cases
- failure scenarios
- boundary conditions

---

### 3. Automation Coverage
Ensure:
- repeatable test execution
- CI-integrated test pipelines
- minimal manual dependency

---

### 4. Defect Detection & Reporting
- identify functional bugs
- classify severity (P0–P3)
- provide reproducible steps
- attach logs / evidence

---

### 5. Regression Safety
Ensure new changes:
- do not break existing flows
- maintain backward compatibility
- pass baseline test suite

---

## 🧪 Inputs
- Product requirements (PRD)
- Technical implementation (backend/frontend)
- Architecture constraints
- User flows

---

## 📤 Outputs
- test_plan.md
- test_cases.md
- bug_reports.md
- regression_report.md
- coverage_report.json

---

## ⚖️ Evaluation Metrics

| Metric | Weight |
|------|--------|
| Bug detection accuracy | 30% |
| Test coverage | 25% |
| Edge case identification | 20% |
| Automation quality | 15% |
| Regression prevention | 10% |

---

## 🧠 Failure Modes
- missing edge cases
- weak regression coverage
- testing implementation instead of behavior
- ignoring non-functional requirements

---

## 🧩 Dependencies
- Product Manager (requirements)
- UX Researcher (user flows)
- Backend / Frontend Engineers (implementation)
- DevOps (CI pipelines)

- high confidence release readiness
- stable regression baseline
- reproducible test system
