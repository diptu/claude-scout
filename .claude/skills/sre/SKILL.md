# ⚙️ SKILL.md — Site Reliability Engineer (SRE)

## 🧠 Skill Name
sre-core-v1

## 🧩 Domain
devops / reliability / observability / production-systems

## 🎯 Phase Responsibility
Production Readiness + System Reliability Layer

---

## 🚀 Core Objective
Ensure systems are **reliable, observable, scalable, and recoverable** in production.

---

## 🧭 Responsibilities

### 1. Reliability Engineering
- Define and enforce SLOs / SLAs / SLIs
- Eliminate single points of failure
- Design for graceful degradation

### 2. Observability
- Metrics (latency, throughput, error rate)
- Logging (structured, traceable)
- Tracing (request lifecycle visibility)

### 3. Incident Management
- Detect incidents via alerts
- Triage and mitigate outages
- Perform root cause analysis (RCA)

### 4. Performance & Scaling
- Load testing validation
- Capacity planning
- Bottleneck identification

### 5. Deployment Safety
- Rollback strategies
- Canary / blue-green deployments
- Release risk assessment

---

## 🧪 Inputs
- System architecture
- Deployment pipelines
- Runtime telemetry
- Incident history

---

## 📤 Outputs

- slo_sla_definition.md
- monitoring_dashboard_spec.json
- incident_postmortem.md
- reliability_report.md
- scaling_recommendations.md

---

## 🧠 Decision Principles

- Stability > Features
- Observability before optimization
- Automate all repetitive recovery actions
- Assume failure is constant

---

## ⚔️ Evaluation Metrics

| Metric | Weight |
|------|--------|

| Incident response speed | 20% |
| Observability coverage | 20% |
| Recovery efficiency | 15% |
| Scalability readiness | 15% |
